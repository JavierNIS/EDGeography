Doxygen admite el uso de la \href{https://www.markdownguide.org/basic-syntax}{\texttt{ sintaxis de Markdown (Consulta aquí los comandos básicos)}}.

Así como el uso de \href{https://htmlcheatsheet.com/}{\texttt{ sintaxis {\bfseries{HTML}}}}. Esto es especialmente útil para aquellos aspectos que Markdown no permite modificar, como el color. Aunque es buena idea evitarlo en la medida de lo posible, por dos motivos\+:


\begin{DoxyEnumerate}
\item Hace el código {\bfseries{poco}}  {\bfseries{ LEGIBLE }} 
\item La idea detrás de Markdown es separar {\itshape aspecto} y {\itshape contenido}. Y es una {\bfseries{gran idea}}.
\end{DoxyEnumerate}

La forma más correcta de modificar el aspecto debería ser mediante un documento {\ttfamily .css} configurado en el archivo {\ttfamily Doxyfile.\+in}. Si te interesa, puedes echarle un vistazo a la \href{https://www.doxygen.nl/manual/customize.html}{\texttt{ documentación}} o a temas customizados como \href{https://jothepro.github.io/doxygen-awesome-css/}{\texttt{ este}}. Aunque el tema por defecto nos parece perfecto, {\itshape CLÁ\+SICO}. Así puedes centrarte en el contenido.

Por último, lo hemos visto en clase y hay suficientes ejemplos en el código fuente proporcionado, pero puedes consultar la mayor parte de la sintaxis de Doxygen para comentar código en su \href{https://www.doxygen.nl/manual/docblocks.html}{\texttt{ documentación}}.\hypertarget{index_autotoc_md1}{}\doxysection{Introducción}\label{index_autotoc_md1}
