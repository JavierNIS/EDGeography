Información sobre Doxygen

Doxygen admite el uso de la sintaxis de Markdown (Consulta aquí los comandos básicos).

Así como el uso de sintaxis HTML. Esto es especialmente útil para aquellos aspectos que Markdown no permite modificar, como el color. Aunque es buena idea evitarlo en la medida de lo posible, por dos motivos:

  1. Hace el código poco LEGIBLE
  2. La idea detrás de Markdown es separar aspecto y contenido. Y es una gran idea.

La forma más correcta de modificar el aspecto debería ser mediante un documento .css configurado en el archivo Doxyfile.in. Si te interesa, puedes echarle un vistazo a la documentación o a temas customizados como este. Aunque el tema por defecto nos parece perfecto, CLÁSICO. Así puedes centrarte en el contenido.

Por último, lo hemos visto en clase y hay suficientes ejemplos en el código fuente proporcionado, pero puedes consultar la mayor parte de la sintaxis de Doxygen para comentar código en su documentación.

Introducción

A continuación se explican los diferentes programas creados para la práctica1 TDA-imagen, explicados y con ejemplos de su funcionamiento

Ejecutables

Negativo:

Calcula el negativo de una imagen

negativo <FichImagenOriginal> <FichImagenDestino>

Parameters
<FichImagenOriginal>Imagen PGM a la que se va a calcular
<FichImagenDestino>Imagen PGM resultado de calcular el negativo

Subimagen:

Corta una imagen

subimagen <FichImagenOriginal> <FichImagenDestino> <Fila> <Columna> <Alto> <Ancho>

Parameters
<FichImagenOriginal>Imagen PGM de la que se va cortar
<FichImagenDestino>Imagen PGM cortada que devuelve el programa
<Fila>Fila donde comienza a cortar la imagen
<Columna>Columna donde comienza a cortar la imagen
<Alto>Número de filas de la imagen cortada
<Ancho>Número de columnas de la imagen cortada
Ejemplo de ejecución

Zoom:

Hace zoom a una imagen

zoom <FichImagenOriginal> <FichImagenDestino> <Fila> <Columna> <Longitudlado>

Parameters
<FichImagenOriginal>Imagen PGM de a la que se le va a hacer zoom
<FichImagenDestino>Imagen PGM con zoom que devuelve el programa
<Fila>Fila donde comienza la imagen para hacer zoom
<Columna>Columna donde comienza la imagen para hacer zoom
<Longitudlado>Lado de la imagen para hacerle zoom
Ejemplo de ejecución

Icono:

Crea un icono a partir de una imagen

icono <FichImagenOriginal> <FichImagenDestino> <FactorEscala>

Parameters
<FichImagenOriginal>Imagen PGM para crear el icono
<FichImagenDestino>Icono devuelto como una Imagen PGM
<FactorEscala>Escala del icono
Ejemplo de ejecución

Contraste:

Calcula el contraste de una imagen

contraste <FichImagenOriginal> <FichImagenDestino> <e1> <e2> <s1> <s2>

Parameters
<FichImagenOriginal>Imagen PGM para calcular el contraste
<FichImagenDestino>Imagen PGM con el contraste aplicado
<e1>Valor mínimo de la imagen introducida
<e2>Valor máximo de la imagen introducida
<s1>Valor mínimo nuevo para la imagen
<s2>Valor máximo nuevo para la imagen
Ejemplo de ejecución

ShuffleRows:

Baraja pseudoaleatoriamente las filas de una imagen

shufleRows <FichImagenOriginal> <FichImagenDestino> <num>

Parameters
<FichImagenOriginal>Imagen PGM para barajar sus filas
<FichImagenDestino>Imagen PGM con las filas barajadas

ShuffleRowsEfic:

Baraja pseudoaleatoriamente las filas de una imagen un numero de veces determinado, lo usamos para calcular la eficiencia de la función shuffleRows alterando la implementación interna

shufleRowsEfic <FichImagenOriginal> <FichImagenDestino> <num>

Parameters
<FichImagenOriginal>Imagen PGM para calcular el contraste
<FichImagenDestino>Imagen PGM con el contraste aplicado
<num>Numero de veces que se intercambian las filas
